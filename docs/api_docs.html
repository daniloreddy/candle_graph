Candle Graph API

Generazione di grafici tecnici OHLCV con Bollinger Bands, RSI e MACD

Panoramica

Il microservizio espone endpoint REST per la generazione di grafici di analisi tecnica su dati OHLCV di criptovalute. Produce immagini PNG con tre pannelli sovrapposti (Prezzo + Bollinger Bands, RSI, MACD) renderizzati interamente in RAM, senza I/O su disco.

Autenticazione: Tutti gli endpoint di generazione grafico richiedono l'header HTTP di autorizzazione Bearer. I token validi sono configurati nel file .env tramite la variabile API_TOKENS (lista separata da virgole). L'health check è pubblico.

Authorization: Bearer INSERISCI_QUI_IL_TUO_TOKEN

Endpoints

GET /health

Verifica lo stato di salute del servizio e la configurazione corrente.

Risposta:

{
  "status": "ok",
  "auth_enabled": true,
  "concurrency_limit": 4
}

POST /api/v1/chart Token

Genera un grafico di analisi tecnica a partire da una serie di candele OHLCV. Restituisce un'immagine PNG o la sua codifica base64.

Body (JSON) — campi principali:

Campo Tipo Default Descrizione
symbol String obbligatorio Ticker del simbolo (es. BTC/USDT). Max 50 caratteri.
data Array obbligatorio Array di candele OHLCV. Min 26 elementi, max 5000. Vedi struttura sotto.
bb_k Float 2.0 Moltiplicatore della deviazione standard per le Bollinger Bands. Deve essere > 0.
max_ohlcv_points Integer 180 Numero massimo di candele da includere nel grafico (ultime N dopo l'ordinamento). Range: 10–1000.
response_format String "png" Formato della risposta: "png" (binario) o "b64" (JSON con stringa base64).

Struttura di ogni elemento in data:

Campo Tipo Descrizione
date String (ISO 8601) Timestamp della candela (es. 2024-01-01T00:00:00).
open Float Prezzo di apertura.
high Float Prezzo massimo.
low Float Prezzo minimo.
close Float Prezzo di chiusura.
volume Float Volume scambiato.

Esempio richiesta:

{
  "symbol": "BTC/USDT",
  "bb_k": 2.0,
  "max_ohlcv_points": 180,
  "response_format": "png",
  "data": [
    {
      "date": "2024-01-01T00:00:00",
      "open": 42000.0,
      "high": 43500.0,
      "low": 41200.0,
      "close": 43100.0,
      "volume": 1850.5
    }
  ]
}

Risposta — response_format: "png":

Binario PNG (Content-Type: image/png).

Risposta — response_format: "b64":

{
  "image_b64": "iVBORw0KGgoAAAANSUhEUgAA..."
}

Errori:

HTTP Causa
400 Dati insufficienti per calcolare gli indicatori (meno di 26 candele effettive dopo lo slice).
401 Token Bearer assente o non valido.
422 Validazione Pydantic fallita (es. symbol > 50 caratteri, max_ohlcv_points fuori range, array > 5000 elementi).
500 Errore interno durante la generazione del grafico.

Dettagli Tecnici

  • Rendering in RAM: Il grafico viene generato interamente in memoria tramite io.BytesIO e FigureCanvasAgg. Nessun file temporaneo viene scritto su disco.
  • Thread-safety: Il servizio utilizza l'API Object-Oriented di Matplotlib (mai pyplot) per evitare corruzione dello stato globale. Ogni richiesta crea la propria istanza Figure isolata.
  • Indicatori calcolati: Bollinger Bands (window 20, moltiplicatore bb_k), RSI(14), MACD(12, 26, 9). Richiedono almeno 26 candele per essere calcolati.
  • Ordinamento e slicing: I dati in input vengono ordinati per date e ridotti alle ultime max_ohlcv_points candele prima del calcolo degli indicatori.
  • Concorrenza: Le operazioni CPU-bound (calcolo indicatori + rendering) vengono eseguite in thread separati tramite asyncio.to_thread e sono limitate a 4 richieste simultanee tramite semaforo.
  • Formato grafico: Dark theme (#0d1117), 3 pannelli sovrapposti: Prezzo con Bollinger Bands (panel 2x), RSI con livelli 30/70 (panel 1x), istogramma MACD con linea signal (panel 1x). Risoluzione 13×11 pollici a 140 DPI.
